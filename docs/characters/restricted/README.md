# Restricted Character Canon

> Index for character material that is canon or planning-canon for development purposes but should not be treated as early reader-facing exposition.

**Canon Status:** MAIN CANON INDEX, RESTRICTED / EXPANDABLE
**Canon Weight:** Index / Character Governance
**Scope:** Restricted character files, reveal discipline, protected backstory material
**Source of Truth:** Yes for restricted character file handling

---

## Purpose

Restricted character files are development-facing lore cabinets.

They may contain:

- hidden backstory
- protected trauma logic
- secret biological or cosmological status
- later-series reveal material
- character motives that should inform behavior before they are explained
- mature horror, institutional horror, or body-horror details

They should not be treated as early Book One exposition.

Core distinction:

**Open canon shows what a character does. Restricted canon explains what systems tried to do to them.**

## Handling Rules

- Do not reveal restricted character truth before the character's open-facing competence is established.
- Do not make secret biology more important than personhood.
- Do not make trauma the character's whole identity.
- Do not let restricted files solve protected mysteries unless explicitly approved.
- Do not make mature horror decorative.
- Do not make another character a therapist, handler, owner, or savior by default.
- Do not confuse development-facing truth with what the crew knows in-scene.

---

## Final Principle

**Restricted canon should sharpen characterization, not spoil the story's teeth before they bite.**
